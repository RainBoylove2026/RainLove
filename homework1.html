+ Understanding Webpage / Web Site / Web Browser

Web page: is a single document or file, typically written in HTML, that is displayed in a web browser.
It’s a component of a website, accessible via a unique URL or file path. (ex: aboutus.html, home.html..)

**Website: A website is a collection of web pages and related content (e.g., images, videos, text)
hosted on a server and accessible via the internet. It’s identified by a domain name (e.g., example.com/facebook.com)
and built using technologies like HTML, CSS, JavaScript, and server-side languages (e.g., PHP, Node.js).

**Web Browser: A web browser is a software application on a user’s device (e.g., computer, phone)
that retrieves, renders, and displays websites from the internet. It interprets the website’s code (HTML, CSS, JavaScript) to present content to the user.

(ex: firefox, google chrome, opera, microsoft edge..)

+ Understanding Static and Dynamic website

Static website

A static website delivers pre-built, fixed content that doesn’t change unless the developer manually updates the code.
Each page is coded in HTML and served to the user exactly as stored.

Content is the same for every user and doesn’t change dynamically.

Typically built with HTML and CSS, sometimes minimal JavaScript for basic interactivity (e.g., image sliders).

Use Cases:

  • Personal portfolios, brochures, or informational sites (e.g., a restaurant’s menu page).
  • Sites with content that rarely changes, like a company’s “About Us” page.

Pros:

  • Simple to create and host.
  • Fast and lightweight.
  • Cost-effective (can be hosted on basic servers like GitHub Pages).

Cons:

  • No real-time interactivity or personalization.
  • Manual updates required for content changes.
  • Not suitable for complex applications.

Example website Static: wikipedia, rfa, dap, filehippo

** Dynamic website

Definition: A dynamic website generates content on-the-fly, often tailored to user input, preferences, or real-time data.
It relies on server-side and/or client-side programming to create interactive, ever-changing experiences.

Often connects to databases (e.g., MySQL, MongoDB) to store/retrieve data like user info or posts.

Supports features like logins, forms, search, or live updates.

Use Cases:

  • Social media platforms (e.g., X, where posts update in real-time).
  • E-commerce sites (e.g., Amazon, with personalized product recommendations).
  • Blogs with comment sections or content management systems (e.g., WordPress).

Pros:

  • Highly interactive and personalized (e.g., user-specific dashboards).
  • Scalable for complex apps (e.g., real-time chats or e-commerce).
  • Supports frequent updates via databases or APIs.

Cons:

  • More complex to develop and maintain.
  • Requires server infrastructure, increasing hosting costs.

Example Dynamic Website: chaingpt.org, facebook, youtube, twitter(x.com), IG...

+ Understanding Front-End (Client-Side) and Back-End (Server-Side) Website:

Front End (Client-Side):

The front end is the part of a website that users see and interact with in their browsers.
It includes everything visible on the webpage, like layouts, buttons, images, and text, as well as the user experience (UX) and user interface (UI).

Example: using facebook or youtube app/website.

Back End (Server-Side):

The back end handles the behind-the-scenes functionality of a website, managing data, servers, and application logic to ensure the front end works smoothly.

Example: the staff or developer of the facebook/youtube company.
